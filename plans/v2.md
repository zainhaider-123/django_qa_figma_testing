# QA Figma Testing Tool — v2 Plan (Simplified)

## Overview
A Django web app that connects to a user's Figma account (OAuth 2.0 via django-allauth),
lets them pick a project → page → frame, enter a live site URL, and runs a pixel-perfect
comparison between the Figma design render and a Playwright screenshot of the live site.
Results are shown in a visual report with mismatch score, side-by-side, and diff overlay.

**v2 philosophy: keep it simple.** SQLite only. No Celery. No Redis. No Postgres.
Screenshot + comparison runs synchronously — the user waits a few seconds and gets
the result. This is a single-user QA tool, not a high-throughput SaaS.

## Is django-allauth the Right Choice for Figma?

**Yes.** django-allauth ships a **built-in Figma OAuth2 provider**
(`allauth.socialaccount.providers.figma`) — already registered in this project.

What allauth gives us for free:
- Full OAuth2 Authorization Code Grant flow (redirect, code exchange, token storage)
- `SocialToken` model stores access_token + refresh_token per user
- `SocialAccount` model stores Figma user id, name, avatar
- Automatic token refresh on expiry (allauth's `OAuth2Adapter` handles refresh_token grant)
- SocialApp configuration via Django admin (no code changes to swap credentials)
- Login required / connection management views out of the box

The alternative — manual OAuth2 with `requests-oauthlib` — would mean reimplementing
token storage, refresh logic, and account linking. Not worth it. allauth is the standard
Django ecosystem choice and the right tool here.

## Figma API Endpoints Used

| Purpose | Endpoint | Scope |
|---|---|---|
| Current user + team_id | `GET /v1/me` | `current_user:read` |
| List team projects | `GET /v1/teams/{team_id}/projects` | `projects:read` |
| List files in project | `GET /v1/projects/{project_id}/files` | `projects:read` |
| File tree (pages + frames) | `GET /v1/files/{file_key}?depth=2` | `file_content:read` |
| Export frame as PNG | `GET /v1/images/{file_key}?ids={node_id}&scale=2&format=png` | `file_content:read` |

### OAuth 2.0 Flow (via django-allauth — built-in Figma provider)
1. User clicks "Connect Figma" → allauth redirects to Figma authorize URL with scopes:
   `file_content:read projects:read current_user:read`
2. Figma redirects back with auth code → allauth exchanges for access_token + refresh_token
3. Tokens stored in `SocialToken` / `SocialAccount` models (allauth-managed)
4. Access token: 90-day lifetime. allauth handles refresh automatically.
5. No custom adapter needed — the built-in `FigmaOAuth2Adapter` handles all URLs and scopes.

## User Flow (Step by Step)

### 1. Connect Figma Account
- User clicks "Connect Figma" → django-allauth OAuth flow → SocialToken stored
- If token expired, allauth auto-refreshes

### 2. Select Project
- `GET /qa/new/` → fetches user's teams → projects → renders project list
- User selects a project → shows files in that project

### 3. Select Page/Frame
- User selects a file → fetch `GET /v1/files/{file_key}?depth=2`
- Render tree: pages (CANVAS) → frames (FRAME) with thumbnails
- User selects a frame → captures `node_id`, `frame_width`, `frame_height`

### 4. Enter Site URL + Run (Synchronous)
- Form: site URL + optional threshold override
- On submit → create `TestRun` (status=running) → **run synchronously**:
  1. Fetch Figma frame PNG → save to `figma_image`
  2. Playwright screenshot of live site → save to `site_screenshot`
  3. Normalize dimensions (resize screenshot to match figma image if needed)
  4. pixelmatch compare → get `mismatch_pixels` + diff image
  5. Compute `mismatch_pct = (mismatch_pixels / (w * h)) * 100`
  6. Determine pass/fail: `pass` if `mismatch_pct <= threshold_pct` (default 1.0%)
  7. Save all images + metrics → status=completed (or failed on exception)
- Redirect to report page
- **User waits ~5-15 seconds** during the run (acceptable for a QA tool)

### 5. View Report
- `GET /qa/run/{id}/` → report page showing:
  - Pass/Fail badge + mismatch percentage (large)
  - Three images side-by-side: Figma design | Live screenshot | Diff overlay
  - Toggle: slider overlay (drag to reveal design vs screenshot)
  - Run metadata: URL, frame name, dimensions, timestamp, threshold
  - "Re-run" and "New Test" buttons

## Setup Requirements
1. `pip install -r requirements.txt`
2. `playwright install chromium`
3. Create Figma OAuth app at figma.com → get client_id + secret
4. Add `FIGMA_CLIENT_ID` + `FIGMA_CLIENT_SECRET` to `.env`
5. `python manage.py migrate`
6. `python manage.py createsuperuser`
7. `python manage.py runserver`
8. Go to `/admin/` → Social Apps → add Figma app with client_id + secret, assign to Site
9. Visit `/accounts/figma/login/` to connect Figma

**No Redis. No Celery worker. No second terminal.** Just `runserver`.

## Risks & Mitigations
| Risk | Mitigation |
|---|---|
| Figma image export URL expires (30 days) | Download + store PNG in TestRun immediately |
| Live site dimensions don't match frame | Resize screenshot to match figma image dims before compare |
| Playwright not installed | Document `playwright install chromium` in setup |
| Token expires mid-session | allauth auto-refreshes; FigmaClient retries on 401 |
| Slow sites cause timeout | Playwright `timeout` config + retry once |
| Anti-aliasing false positives | pixelmatch handles AA by default (configurable) |
| Synchronous run blocks request ~15s | Acceptable for QA tool; show loading spinner; add Celery later if needed |
